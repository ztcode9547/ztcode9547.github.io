网络通信：Socket+IO复用+多线程或者线程池+libevent框架

基本概念

局域网

覆盖范围较小的计算机网络，外面访问不到。

广域网

是一种覆盖广泛地理区域的计算机网络，它可以跨越城市、国家甚至跨越大陆。

网络协议

是互相通信对等实体必须遵守的一种规则，这种规则可以是标准的，也可以说自定义的。

IP协议

  • 是表示终端在网络中的地址，分为IPv4和IPv6。

  • 我们在window系统cmd下输入命令ipconfig或者在Linux系统下输入ifconfig都可以显示ip地址，这个ip地址是局域网内的ip地址，如果使用网线接入外网，会统一分配公网IP地址。

  • IPv4本质是一个整形数，有4个字节，所以有32位，所以有2的32次方个ip地址。

  • IPv4使用点分十进制来表示，比如192.168.247.135，最小为：0.0.0.0，最大为：255.255.255.255。

  • 可以使用ping+域名的方式查看当前设备是否能访问域名地址。

端口

  • ip地址是能够定位设备，端口是能够定位计算机里面的进程。

  • 端口本质是一个无符号的短整型，占两个字节，所以有0到65535种可能性。

  • 客户端执行connect函数后，会分配一个新的端口。

Tcp/Ip四层模型

应用层：最上层，也是程序员写业务的层次。有HTTP（超文本传输协议），FTP（文件传输协议）

传输层：TCP协议，UDP协议

网络层：IP协议

网络接口层：无线局域网（WLAN），以太网（Ethernet）

字节序

  • 顾名思义是字节的顺序，就是大于一个字节类型的数据在内存中的存放顺序，也就是说对于单字符来说是没有字节序问题的，字符串是单字符的集合，因此字符串也没有字节序问题。

  • 目前字节序有两种：小端字节序（主机字节序）和大端字节序（网络字节序）

  • 小端字节序（主机字节序）：

    • 数据的地位存储到内存的地位，数据的高位存储到内存的高位。
    • 我们使用的PC机，数据的存储默认使用的是小端。

  • 大端字节序（网络字节序）：

    • 数据的地位存储内存的高位，数据的高位存储内存的地位。
    • 套接字通信过程中操作的数据都是大端存储的，包括：接收/发送的数据、IP地址、端口。

  • 字节序在编程中一般在ip地址和端口绑定需要手动转换，而在发送数据和接收数据的函数里，系统内核自动帮我们进行字节序的转换

总结:我们使用的设备是用主机字节序的，而网络通信是使用网络字节序的，所以我们要把数据转为网络字节序进行通信，通信结束，然后又把网络字节序转为主机字节序进行接收。

Socket网络通信

​ 就像一台排插，排插就是服务器，插头就是客户端，排插有多个插口，等待多个插头链接。和套接字相关的函数被包含在头文件arpa/inet.h中。一个头文件就全部包含了socket相关的api。

TCP通信流程

TCP三大特点：面向连接，安全，流式传输

面向连接：是一个全双工的通信，三次握手建立连接，四次挥手解除链接。

安全：会对每一个包进行校验，丢包会重发。

流式传输：相当于发送接收两端有一条水管，发送多了，接收不了那么多，会让发送慢点。

文件描述符在网络通信中的作用

  • 一个文件描述符对应两个内存。一个叫读缓冲区，一个叫写缓冲区。我们在调用accept函数，send函数和wirte函数时，客户端和服务器不是直接发送的，而是通过这些函数往这些缓冲区写数据或者读数据，然后内核会帮我们从缓冲区里面拿数据进行网络通信，是操作系统自动的。

  • 一个文件描述符对应两个内存：一个叫读缓冲区，一个叫写缓冲区。

    • 读缓冲区：通过文件描述符可以把内存的数据读出。

    • 写缓冲区：通过文件描述符可以往内存写入数据。

  • 监听类型的文件描述符：也包含两块内存

    • 调用accpet函数会监听这个文件描述符的读缓冲区，如果没有连接请求，这个读缓冲区为空，则会阻塞。这也是调用accept函数会阻塞的原因。
    • 如果不为空，则可以往读缓冲区内存读数据，则解除阻塞。说明有新的客户端连接。

  • 通信的文件描述符:也包含两块内存

    • 和上面类似，调用的send函数和wirte函数都是监听内存是否有数据。

总结：通过这种方式，可以增加网络通信的安全性，程序员不必管数据是如何发送和接收的，只要把数据放入相应的缓冲区和读取相关的缓冲区即可。

#include"servero.h"
#include<string.h>
#include<stdio.h>
#include<stdlib.h>
#include<unistd.h>
#include<sys/socket.h>
#include<arpa/inet.h>
int echo_server()
{
	//创建套接字
	int serfd = socket(AF_INET, SOCK_STREAM, 0);
	if (serfd == -1) {
		perror("socket create fail\n");
		return -1;
	}
	//捆绑ip和端口
	struct sockaddr_in serAddr;
	serAddr.sin_addr.s_addr = INADDR_ANY;//这个宏的值为0 == 0.0.0.0，可以接受任何地址来的客户端
	serAddr.sin_family = AF_INET;
	serAddr.sin_port = htons(10000);//htons函数将端口主机字节序-》网络字节序
    int ret = bind(serfd, (struct sockaddr*)&serAddr, sizeof(serAddr));
	if (ret == -1) {
		perror("bind fail\n");
		return -1;
	}
	//listen函数
	ret = listen(serfd, 5);
	if (ret == -1) {
		perror("listen fail\n");
		return -1;
	}
	//accept函数
	struct sockaddr_in cliAddr;
    int clilen = sizeof(cliAddr);
	int cliefd = accept(serfd, (struct sockaddr*)&cliAddr, &clilen);//和bind不一样，accept存的是指向内存大小那个变量的指针
	if (cliefd == -1) {
		perror("accept fail\n");
		return -1;
	}
	printf("客户端的ip地址是%s,端口号为%d\n", inet_ntoa(cliAddr.sin_addr), ntohs(cliAddr.sin_port));//打印客户端的地址和端口
	while (1) {
	//read函数
		char* buf = malloc(sizeof(char) * 1024);
		memset(buf, 0, sizeof(char) * 1024);
		ssize_t read_len = read(cliefd, buf, sizeof(buf) * 1024);
		if (read_len > 0) {
			//send函数
			printf("客户端说：%s", buf);
			ssize_t write_len =write(cliefd, buf, read_len);//发送不需要发送这个buf，看读到多少字节，或者使用stlen函数+1。
			if (write_len == -1) {
				perror("发送数据失败了\n");
				return -1;
			}
		}
		else if (read_len == 0) {
			perror("客户端断开连接\n");
			return -1;
		}
		else {
			perror("接收数据失败\n");
			return -1;
		}
		free(buf);//动态分配内存记得释放
	}
	//close函数
	close(cliefd);
	close(serfd);
};

缺点：服务器只能和一个客户端通信，且服务器没有客户端连接会阻塞。连接后客户端没有发数据，服务器也会阻塞。基于上述处理方式，在单线程/单进程场景下，服务器是无法处理多连接的。

改进：使用多线程；使用多进程；使用IO多路复用；多线程+IO多路复用。

遇到的一个坑：使用printf函数打印中文，文件一定要使用utf-8格式，Linux系统是默认使用utf-8格式，不然会出现乱码。解决方法：使用记事本把文件另存为utf-8格式，然后替代原来的文件。

#include <cstdio>
#include<string.h>
#include<stdio.h>
#include<stdlib.h>
#include<unistd.h>
#include<sys/socket.h>
#include<arpa/inet.h>
int main()
{
    //创建socket
    int clifd = socket(AF_INET, SOCK_STREAM, 0);
    if (clifd == -1) {
        perror("socket create fail\n");
        return -1;
    }
    
    //connect函数
    struct sockaddr_in serAddr;
    serAddr.sin_addr.s_addr = inet_addr("192.168.136.1");
    serAddr.sin_family = AF_INET;
    serAddr.sin_port = htons(10000);
    int ret = connect(clifd, (struct sockaddr*)&serAddr, sizeof(serAddr));
    if (ret == -1) {
        perror("connect fail\n");
        return -1;
    }
    int num = 0;
    while (1) {
      //send函数
        char buf[1024];
        printf("%d\n", num);
        sprintf(buf, "第%d次说话\n", num++);//动态格式化字符串
        sleep(1);//延迟1秒
        ssize_t send_len = send(clifd, buf, strlen(buf)+1, 0);//不使用sizeof，不必发送整个buf，使用strlen函数+1看字符串长度。
        memset(buf, 0, strlen(buf));
        if (send_len == -1) {
            perror("发送数据失败了\n");
            return -1;
        }
      //read函数
        ssize_t raed_len = read(clifd, buf, sizeof(buf));
        printf("%s", buf);
        if (raed_len == -1) {
            perror("接收数据失败了\n");
            return -1;
        }
    }
   //close函数
    close(clifd);
    return 0;
}
